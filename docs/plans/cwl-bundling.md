# CWL bundling — implementation plan (DP-9)

Build plan for design-spec §10 **DP-9** (the §8.5 CWL row, resolved
2026-07-16). Data distribution only: no engine, CLI-flag, or default
change. Evidence backing every choice here:
[`../../benchmarks/fill_quality/README.md`](../../benchmarks/fill_quality/README.md),
"CWL on the §8.4c engine".

  commit `2efe76e11ef311315e76d59700752733d69733d7`
  (`xwordlist.dict` last changed 2023-02-12; MIT re-verified at source).
- Raw `xwordlist.dict` sha256:
  `a945a839a5f1e6f48caf9c8de446e5cd85f3567d7f62afcf54c6b738e8906ff4`

- Shipped derivative `dicts/cwl50.dict`: drop lines whose word field

## Deliverables

1. **`scripts/fetch-cwl.sh`** — fetches the pinned raw file (commit URL,
   never a branch), **verifies the raw sha256 before doing anything else**,
   digit-filters, applies `--min-score N` (default 50), writes `--out FILE`
   (default `dicts/cwl50.dict`). Warns for `N < 50` (the ≥30 band loads but
   crashes §8.4c search on full-13 grids; `N < 30` approaches the untested
   load ceiling; the unfiltered list cannot load at the default stack).
   The default invocation must regenerate the bundled artifact
   **byte-identically**.
2. **`dicts/`** (new, product asset dir like `grids/`):
   `cwl50.dict` (generated by 1), `CWL-LICENSE` (upstream MIT text
   verbatim), `README.md` (provenance + both sha256s + filter rationale +
   usage: no `--min-score` needed, 75/90 are the stricter in-file bands;
   staleness note — frozen 2023-02-12; honesty note — does not fill the UK
   stock grids, pair with `grids/amer11.json`).
3. **`grids/amer11.json`** — promote the benchmark's authored American
   11×11 (OD-6 "set grows"; mask PASSes `stockgrid_report`, its cwl50 fill
   PASSes `lint --profile blocked-uk`). Add the `grids/README.md` row and a
   `bundled_grid('grids/amer11.json')` fact in `tests/stockgrid.plt`.
4. **Tests** — (a) plunit smoke in `tests/fill.plt`: load `dicts/cwl50.dict`
   via the module's own loader; assert 252,200 words, every score ≥ 50, a
   spot word/score; (b) golden `fill_cwl_amer11`: the README demo command
   (`fill --grid grids/amer11.json --dict dicts/cwl50.dict`), byte-pinned
   like the other fill goldens (~6.4s — accepted `make test` cost, it is
   the end-to-end smoke of the shipped pairing).
5. **Docs** — README.md (fill quick-start example uses the bundled pairing;
   `--dict` row mentions `dicts/cwl50.dict`; the "no scored list is
   bundled" limitation bullet rewritten; capacity envelope noted),
   `docs/STATUS.md` (DP-9 resolved + built).

## Non-goals (explicit)

- No default-dict change: existing goldens, `fill_identity.sha256`, and
  both ratchets must be **untouched** (verified by `make test` passing
  without regeneration).
- No engine change: the two capacity overflows stay a §8.5 backlog row.
- No wasm change: `dicts/` is not staged into the wasm payload.
- STW stays point-at-only (CC BY-NC-SA — never committed).

## Verification

`make test` (plunit + goldens + CLI exit codes, incl. the new smoke +
golden + amer11 stockgrid fact), `make fuzz`. Identity/ratchet baselines:
diff-clean. `scripts/fetch-cwl.sh` run once against the network to prove
byte-identical regeneration of the committed artifact.
